# RivalScope API Docs

## Назначение

RivalScope анализирует конкурентов в выбранной нише. Основной сценарий ДЗ: выбрать 3-5 конкурентов, собрать материалы в `data/`, проанализировать сайты/тексты/изображения и подготовить выводы по рынку.

## Пайплайн

1. Пользователь выбирает нишу или вводит свою.
2. Пользователь указывает свой бренд и добавляет конкурентов.
3. Пользователь отправляет текст, изображение или URL.
4. Backend валидирует запрос.
5. Для сайта Selenium открывает страницу и собирает HTML/скриншот.
6. AI-сервис отправляет данные в ProxyAPI.
7. Модель возвращает строгий JSON.
8. Результат сохраняется в `history.json`.
9. UI показывает оценочную карту, SWOT, таблицу сравнения или action plan.

## Endpoints

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/` | Веб-интерфейс |
| `GET` | `/health` | Проверка backend и AI-провайдера |
| `GET` | `/competitors` | Демо-набор конкурентов |
| `POST` | `/analyze_text` | Анализ текста конкурента |
| `POST` | `/analyze_image` | Анализ изображения |
| `POST` | `/parse_demo` | Анализ сайта через Selenium/http fallback |
| `POST` | `/compare_competitors` | Сравнительный AI-вывод |
| `POST` | `/analyze_pdf` | Заготовка под расширение PDF |
| `GET` | `/history` | История запросов |
| `DELETE` | `/history` | Очистка истории |

## Пример `/analyze_text`

```json
{
  "text": "Внедряем AI-агентов и n8n автоматизации для бизнеса...",
  "context": {
    "niche": "AI-автоматизация бизнеса",
    "custom_niche": null,
    "own_brand": "NeiroBridge",
    "own_site": "https://neirobridge.ru",
    "target_audience": "Малый и средний бизнес"
  }
}
```

## Формат AI-анализа

Ожидаемые поля для текста/сайта:

- `summary` — краткий вывод.
- `strengths` — сильные стороны.
- `weaknesses` — слабые стороны.
- `unique_offers` — УТП.
- `recommendations` — рекомендации.
- `scores` — оценки 0-10.
- `positioning_gap` — разрыв позиционирования.
- `action_plan` — практические шаги.

Для режима сравнения возвращается отдельная структура:

- `competitor_cards` — карточки конкурентов.
- `comparison_table` — таблица: критерий, мой бренд, конкуренты, что улучшить.
- `brand_recommendations` — рекомендации для своего бренда.
- `quick_wins` — быстрые улучшения.
- `strategic_moves` — стратегические шаги.
- `risks` — риски.

Шкалы отличаются по типам анализа:

- текст: оффер, отличие от конкурентов, доверие, понятность, конверсия, релевантность автоматизации;
- сайт: оффер, дизайн, доверие, понятность, CTA, фокус на автоматизации;
- изображение: визуальный стиль, ясность, доверие, CTA, потенциал анимации.

## Расширения

- PDF-анализ через `PyMuPDF`.
- Экспорт отчета в `reports/`.
- GigaChat fallback при недоступности ProxyAPI.
- Полный desktop-анализ изображений.
