# Отчет по домашнему заданию: RivalScope

## 1. Идея проекта

RivalScope — персональный AI-анализатор рынка конкурентов. Приложение помогает пользователю выбрать нишу, добавить свой бренд и конкурентов, затем проанализировать сайты, тексты и изображения конкурентов.

Основной кейс для тестирования: рынок AI-автоматизации бизнеса, n8n-сценариев, AI-агентов и чат-ботов.

## 2. Ниша и тестовые конкуренты

Для тестирования использованы:

- `https://neirobridge.ru` — сайт моего бренда.
- `https://aiworkflows.ru` — конкурент в AI-автоматизации, RAG, n8n и CRM.
- `https://www.4gic.com` — конкурент с фокусом на AI-агентов, консалтинг и ROI.
- `https://cortex-lab.ru` — конкурент с фокусом на n8n, Telegram/WhatsApp-ботов и voice AI.

При этом приложение не привязано к этим сайтам: пользователь может выбрать любую нишу, указать свой бренд и добавить собственных конкурентов.

## 3. Реализованный функционал

- Веб-интерфейс в стиле Strategic Intelligence.
- Выбор ниши из списка или ввод своей ниши.
- Поля для своего бренда, сайта бренда и целевой аудитории.
- Динамический список конкурентов.
- Анализ сайта через Selenium с fallback на HTTP-парсинг.
- Анализ текста конкурента.
- Анализ изображения через vision-модель.
- Режим сравнения конкурентов: карточки, таблица, рекомендации, быстрые улучшения и стратегические шаги.
- История запросов в `history.json` с просмотром сохраненных результатов.
- Progress bar и блокировка кнопок во время долгих операций.
- Desktop-приложение на PyQt6.
- Сборка desktop-приложения в `.exe` через PyInstaller.
- README и API-документация.

## 4. Технологический стек

- `Python` — основной язык backend и desktop.
- `FastAPI` — API и веб-сервер.
- `Uvicorn` — запуск backend.
- `Pydantic` — схемы запросов и ответов.
- `ProxyAPI` — OpenAI-совместимый доступ к моделям.
- `gpt-4o-mini` — быстрый анализ текста и сайтов.
- `gpt-4o` — анализ изображений и более глубокие отчеты.
- `Selenium` — открытие сайтов через Chrome и сбор данных.
- `BeautifulSoup4` — извлечение текста из HTML.
- `HTML/CSS/Vanilla JS` — веб-интерфейс.
- `PyQt6` — desktop-клиент.
- `PyInstaller` — сборка `.exe`.

## 5. Pipeline приложения

1. Пользователь выбирает нишу и вводит контекст своего бренда.
2. Пользователь добавляет конкурентов или использует демо-набор.
3. Пользователь запускает анализ сайта, текста, изображения или сравнение.
4. Backend валидирует запрос.
5. Для сайта Selenium открывает страницу, получает HTML и скриншот.
6. AI-сервис отправляет данные в ProxyAPI.
7. Модель возвращает структурированный JSON.
8. Backend сохраняет результат в историю.
9. Frontend показывает отчет: оценки, сильные/слабые стороны, таблицы и рекомендации.

## 6. Что заложено как расширение

- PDF-анализ через `PyMuPDF`.
- Fallback на GigaChat при недоступности ProxyAPI.
- Экспорт финального отчета в `reports/`.
- Более полная desktop-версия с анализом изображений.

## 7. Безопасность

API-ключи не хранятся в коде. Для ключей используется `.env`, который добавлен в `.gitignore`.
